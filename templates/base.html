{# IMPORTANT STATIC PATH STRATEGY We intentionally use *relative* paths (no leading slash) for static assets: href="static/css/style.css" and src="static/js/main.js" Why: Nginx rewrites /gid/ -> / before hitting FastAPI. The application cannot reliably detect the original /gid prefix (request.url.path is already rewritten), so url_for('static', ...) would emit /static/... which in production maps to a different upstream (n8n) and breaks CSS/JS loading. Relative paths allow the browser to resolve against the *original* URL the user sees. - When accessed at https://domain/gid/, "static/css/style.css" resolves to /gid/static/css/style.css which Nginx rewrites to /static/css/style.css and FastAPI serves correctly. - When accessed locally at http://localhost:8000/ the same relative path becomes /static/css/style.css. - When accessing locally at http://localhost:8000/gid/ (sub-app) it becomes /gid/static/css/style.css which is mounted as well. This keeps one portable template for all environments without needing nginx changes. #} {% block styles %}{% endblock %}
